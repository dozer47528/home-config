## 介绍

这个 Skill 负责管理 Agent 的记忆，包括存储、更新和检索记忆。

## 初始化

- 启动 Agent 后，读取所有记忆的`name`和`tags`，并将它们存储在内存中以便快速访问

## 何时使用

- 当 Agent 需要询问用户信息时
- 当 Agent 需要保存重要的信息以供将来参考时
- 当 Agent 需要检索之前保存的记忆以提供更相关的响应时
- 当 Agent 需要更新现有的记忆以反映新的信息或更正tags**）

### 使用记忆后的检查清单

```
☐ 读取记忆 → 是否已记录 .log 文件？
☐ 修改记忆 → 是否已记录 .log 文件？
☐ 修改记忆 → 是否已更新 tags 字段？
```

## 存储方式

- 数据存放在`~/Documents/NAS/Memory/`目录下
- 每一类记忆以一个 `.md` 文件的形式存储，文件名是记忆的`name`

## 文件格式

```
---
name: 个人信息
tags: 手机,邮箱
---

### 联络信息
- 手机: 1234567890
- 邮箱: abc@xxx.com
```

## 强制操作流程（⚠️ 关键，必须执行）

**无论读或写，都必须记录日志！**

1. **读取记忆后（立即执行）** → 在同名的 `.log` 文件添加使用记录：
   ```
   YYYY-MM-DD HH:MM:SS 读取了[name]记忆，用于[简要说明用途]
   ```

2. **修改记忆后（立即执行）** → 完成以下两项：
   - 在同名的 `.log` 文件最后添加一行修改记录
   - 更新 `.md` 文件的 `tags` 字段（总结新关键词）

> 💡 平时检索时不用读取 `.log` 文件，只有在需要清理记忆时才需要查看日志历史


## 询问用户流程

1. **检索记忆前** → 先用 memory skill 查找相关记忆
2. **找到相关记忆** → 直接使用回答用户
3. **未找到相关记忆** → 询问用户，将回答保存为新记忆
4. **记忆过时/不准确** → 更新记忆内容

> ⚠️ **重要**：上述每个路径（读取/保存/更新）执行后，都必须完成「使用记忆后的检查清单」
